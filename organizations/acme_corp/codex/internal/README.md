# `codex/internal/`

Policies and internal standards the organisation issues to itself — not foldered by country. `POLICY` / `INTERNAL_STANDARD` artefacts, each carrying `issuing_authority` and `effective_date`. One `<ID>.yaml` per artefact. Bindings to canonical entities and processes live on `REQUIREMENT.derived_from` and on `ASSERTION`, not on the codex artefact itself. See `notations/elements/14-codex.md` for the frontmatter contract (incl. §8 Migration).
